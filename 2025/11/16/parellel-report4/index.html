二、实验目标与问题描述

本次实验的目标是在 MPI + CUDA 混合并行环境 下，加速如下 GF(2)（模 2）意义上的矩阵迭代计算：
$$
C ;=; (A ;+; \sum_{t=1}^{m} A B^t) \pmod{2},
$$
其中：

  • $A, B \in {0,1}^{n \times n}$；
  • 所有加法、乘法均在模 2 下进行，则：
    • 乘法：$x \cdot y$ 等价于 x & y；
    • 加法：$(x + y) \bmod 2$ 等价于 x ^ y。

给定模板代码中串行实现的基本结构为：

  1. 维护 $B^{(t)}$ 与 $B^{(t+1)}$；
  2. 对于每一轮 $t = 1..m$：
    • 计算 $D = A \cdot B^{(t)}$，并令 $C \leftarrow (C + D) \pmod{2}$；
    • 若 $t < m$，再计算 $B^{(t+1)} = B^{(t)} \cdot B$。

该串行算法时间复杂度为 $O(m n^3)$，在 $n, m$ 较大时计算量巨大。

三、整体并行与优化思路

在保持题目接口不变的前提下，本实验从以下三个层面进行优化：

  1. GPU 端：使用 Tile + Shared Memory 的矩阵乘法内核
    • 将矩阵划分为 TILE × TILE 的子块，在共享内存中缓存 A、B 的局部子块，减少全局显存访问，提高矩阵乘法的算力利用率。
  2. MPI 端：按行划分的分布式 $B^{(t)}$ 幂运算
    • 将 $B^{(t)}$ 的行按 rank 划分，每个进程只负责计算自己对应行块的 $B^{(t+1)}$，利用 MPI_Iallgatherv 将各行块拼成完整 $B^{(t+1)}$，从而在 B 的幂运算上实现 多进程加速。
  3. 通信–计算重叠
    • 利用 MPI_Iallgatherv 发起 $B^{(t+1)}$ 的非阻塞 Allgather；
    • 同时在 GPU 上用当前的 $B^{(t)}$ 计算 $A B^{(t)}$ 并更新 C；
    • 通过将通信与计算交叠，减少纯等待时间。

整体思路是：在行划分的基础上，同时发挥 GPU并行计算 + 多进程分布式计算 + 通信与计算重叠 三种并行性。

四、具体实现方案

1 MPI 层：A 行划分与分布式 B 幂运算

1.1 A 的行划分与分发

  • 使用简单的「商 + 余数」策略将行均匀划分：
    • 第 rem = n % P 个进程之前的 rank 每个多拿一行；
    • 通过函数 compute_row_partition(n, P, rank, &offset, &local_rows) 统一计算每个 rank 的起始行与行数。
  • 矩阵 A 的分发与结果 C 的收集：
    • 使用 MPI_Scatterv 将 A 的行块分发到各个进程，得到 A_local；
    • 最终用 MPI_Gatherv 将各 C_local 拼接成完整 C。

1.2 B 与 B_power 的分布式幂运算

为了避免每个 rank 完整计算一次 $B^{(t+1)}$，对 B 的幂运算也做行划分：

  • 将 $B^{(t)}$ 的行按相同方式划分为若干行块；

  • 在每轮迭代中：

    1. 每个 rank 仅计算自己负责的行：
      $$
      B^{(t+1)}{\text{local}} = B^{(t)}{\text{local}} \cdot B
      $$

    2. 使用 MPI_Iallgatherv 将 B_next_local 收集为完整的 B_next；

    3. 交换指针 B_power ↔ B_next，供下一轮迭代使用。

这样，B 的幂运算计算量从原始的 $O(n^3)$ 变为 每个 rank 约 $O(n^3 / p)$，整体计算效率提升($p$为进程数)。

2 CUDA 层：Tile + Shared Memory 的 GF(2) 矩阵乘法

2.1 Kernel 设计

在 GPU 端矩阵乘法采用 TILE × TILE（如 16×16） tile 的 shared memory kernel：

  • 每个 thread block 对应 C 中的一个 TILE × TILE 子块；
  • 内部使用两个 __shared__ tile：
    • As[TILE][TILE] ：缓存 A 的一个子块；
    • Bs[TILE][TILE] ：缓存 B 的一个子块。

核心循环结构：

  1. 按 K 方向遍历 tile：
    • 每次从全局内存加载一块 A、B 的子块到共享内存；
    • block 内所有线程 __syncthreads() 后复用该 tile。
  2. 对 tile 内进行 TILE 次乘加：
    • 元素乘法 A×B 用 &；
    • 在寄存器中累加整数 tmp；
  3. 最终写回时，做一次 tmp & 1 得到 GF(2) 结果。

述 kernel 同时被用于：

  • 本地 $D = A_{\text{local}} \cdot B^{(t)}$；
  • B 幂运算中的 $B^{(t+1)}{\text{local}} = B^{(t)}{\text{local}} \cdot B$。

3 通信–计算重叠实现

在第 $t$ 轮迭代中，存在两类计算：

  • 对 C：需要当前的 $B^{(t)}$，计算 $A B^{(t)}$；
  • 对 B：需要当前的 $B^{(t)}$，计算 $B^{(t+1)} = B^{(t)} B$。

依赖关系允许我们按如下顺序安排操作：

  1. 在每个 rank 上，基于当前 B_power 计算本地 B_next_local：
    $$
    B^{(t+1)}{\text{local}} = B^{(t)}{\text{local}} B
    $$

  2. 立即调用 MPI_Iallgatherv（非阻塞）将 B_next_local 发起全局收集（通信）；

  3. 与此同时，在 GPU 上用当前 B_power 计算：
    $$
    D_{\text{local}} = A_{\text{local}} B^{(t)}
    $$
    并执行 C_local ^= D_local；

  4. 计算完成后，调用 MPI_Wait，确保 B_next 已完整收集；

  5. 交换指针：B_power ↔ B_next，进入下一轮。

通过这一调度，B 的通信阶段与 A×B 的计算阶段尽可能重叠，减少进程被动等待通信完成的时间。

五、实验结果与分析

  1. 与串行版本对比：

    数据集 small_1 small_2 small_3 large_1 large_2 large_3
    加速比 2.24e-5 6.52e-5 7.61e-5 12.39 68.98 62.05
    • 在小矩阵规模（如 $n \le 256$）下，MPI + CUDA 的固定开销（进程启动、内存拷贝等）较大，整体加速效果有限，甚至更慢；
    • 随着矩阵规模增大，单轮计算量大幅增加，GPU 与多进程优势开始体现，整体加速比可以达到数量级提升。

  2. 与未优化的 MPI+CUDA 初始版本对比：

    数据集 small_1 small_2 small_3 large_1 large_2 large_3
    优化后加速比 2.24e-5 6.52e-5 7.61e-5 12.39 68.98 **62.05
    原始加速比 1.91e-5 6.20e-5 7.59e-5 11.63 16.15 27.96
    • Tile + Shared Memory 内核显著降低了单次矩阵乘的 GPU 时间；
    • 分布式 B 幂运算明显减少了 B 更新部分的重复计算，尤其在较大 m、较大 p 时更为突出；
    • 引入通信–计算重叠后，Allgather 的显式开销得到部分隐藏，迭代总时间进一步缩短。

  3. 扩展性与瓶颈：

    • 随着进程数 p 增大，B 幂运算的计算部分大致按 1/p 缩减，但 Allgather 通信开销逐渐成为瓶颈；
    • 当 A×B 计算本身很快（例如强 GPU + 中等 n）时，通信占比上升，继续提高 p 的收益减弱；
    • 通信–计算重叠在此起到“平滑”网络开销的作用，但无法完全消除通信瓶颈。

整体来看，最终版本较串行和 naive 并行基线都取得了明显的性能提升。

七、实验总结

本次实验围绕 GF(2) 矩阵迭代运算，从串行基线出发，逐步引入：

  1. MPI 行划分：将 A 与 B 的幂运算按行拆分到不同进程上，发挥多进程的并行能力；
  2. CUDA Tile + Shared Memory：在 GPU 上优化矩阵乘法访存，提高单次算子的效率；
  3. 分布式 B 幂运算：避免 B 幂在各 rank 重复计算，实现 B 更新部分的多进程加速；
  4. 通信–计算重叠：通过 MPI_Iallgatherv 将 B 的 Allgather 与 A×B 的计算交叠，减少通信等待。

最终得到的方案在保证数值正确性的前提下，实现了 设备内并行 + 设备间并行 + 通信重叠 的综合优化，后续可引入 bit-packing等更激进的量化优化方案。
